# Beginner Tooltip Revision

## Goal

Make help text understandable for a first-time trainee with no need to infer what the field expects.

## Revision Rules

Each tooltip now answers three things in plain language:

- what should I write here
- how specific should I be
- one short example

## Revised Fields

### Session goal

- now asks for one short learning target
- removes abstract wording like "focused session goal"
- gives a concrete example about identifying one automatic thought

### Tool rationale

- now asks why the selected tool fits this case right now
- makes the goal-to-tool link explicit
- gives a direct thought-record example

### Step input

- now tells the trainee to write the actual CBT response, not notes
- tells them to stay specific to the case
- gives a one-line example of a usable thought statement

### Session summary

- now tells the trainee to write 1 to 2 sentences only
- clarifies that the main point matters more than every detail
- gives a short example of what a useful summary sounds like

### Homework

- now tells the trainee to assign one small realistic task
- makes the tool-link requirement explicit
- gives a short one-task example

### Official Review

- now says this is the first section to read
- explains that it gives the official result and what must be fixed

### Advisory Guidance

- now says this comes after official review
- explains that it helps learning but does not change the result

### Next action

- now emphasizes priority and order
- gives an example of a concrete revision action

### Next focus

- now explains that it is optional and broader than next action
- gives a short example that distinguishes practice focus from required correction

## Result

The help text is now less expert-facing and more instructional.
