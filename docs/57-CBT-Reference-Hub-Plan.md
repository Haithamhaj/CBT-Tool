# CBT Reference Hub Plan

## Purpose
The CBT Reference Hub is the product's main study and grounding page. It is not a marketing overview and not a dump of notes. It is a practical reference layer trainees can return to when they need:

- a quick reminder of the CBT flow
- a simpler explanation of a concept
- a structured map of the early training path
- a tool reminder before or after practice

## Core vs Advanced Split

### Core Reference
Shown first and kept open by default. This is the trainee-facing essential layer.

Includes:
- what CBT is
- core CBT principles
- fixed session structure
- 5-session exploration map
- tools library
- levels of thinking
- distortions reference
- emotion reference
- worksheets/templates reference
- goals and treatment planning overview
- glossary

### Advanced Reference
Shown second and visually secondary through collapsible sections.

Includes:
- deeper cognitive formulation breakdown
- triggering and predisposing factors
- severity and assessment references
- working hypotheses
- limited diagnostic/assessment reference

## Language Handling
- The page uses the same app language selector already used across the shell.
- Arabic mode renders `rtl` and English mode renders `ltr`.
- All hub chrome and all hub content are stored directly in Arabic and English to avoid mixed-language sections.
- The hub should feel fully single-language within the page.

## UX Direction
- top-level tabs instead of long-scroll-only navigation
- quick-start entry layer for common use cases
- short sections with operational summaries
- diagrams for the CBT model, session flow, and five-session progression
- checklist-style content where practical
- collapsible advanced material
- no dense essay blocks unless needed for a definition

## Delivery
- one new route: `/reference`
- one new navigation item
- one bilingual content source file
- lightweight styles only for this page
